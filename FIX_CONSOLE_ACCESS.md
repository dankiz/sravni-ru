# Восстановление доступа к консоли сервера

## Проблема: Не могу подключиться к консоли

## Решения:

### 1. Проверьте веб-консоль RUVDS

- Зайдите на https://console-a.ruvds.com
- Войдите в панель управления
- Найдите ваш сервер (RU2289652)
- Попробуйте открыть веб-консоль через панель управления

### 2. Используйте SSH напрямую (если есть доступ)

На вашем компьютере (Windows PowerShell или CMD):

```bash
ssh root@195.58.51.50
```

Или используйте PuTTY:
- Host: 195.58.51.50
- Port: 22
- Username: root
- Password: (ваш пароль)

### 3. Проверьте статус сервера в панели RUVDS

1. Зайдите на https://ruvds.com/ru-rub/my/servers
2. Проверьте статус сервера
3. Убедитесь, что сервер включен
4. Попробуйте перезагрузить сервер через панель

### 4. Восстановление через панель управления RUVDS

В панели управления RUVDS:
- Найдите раздел "Консоль" или "VNC"
- Попробуйте открыть консоль через другой метод доступа
- Проверьте, не заблокирован ли доступ

### 5. Проверка через панель управления

В панели RUVDS:
- Сервер → Консоль
- Или Сервер → VNC
- Или Сервер → SSH доступ

### 6. Если сервер не отвечает

Возможные причины:
- Сервер перегружен (502 ошибка могла быть из-за этого)
- Сервер завис
- Проблемы с сетью

**Решение:**
- Перезагрузите сервер через панель управления RUVDS
- Подождите 2-3 минуты
- Попробуйте подключиться снова

### 7. Альтернативные способы доступа

#### Через WinSCP (SFTP):
1. Скачайте WinSCP
2. Подключитесь по SFTP:
   - Host: 195.58.51.50
   - Port: 22
   - Username: root
   - Password: (ваш пароль)
3. Это даст доступ к файлам, но не к консоли

#### Через FileZilla:
- Аналогично WinSCP

### 8. Проверка доступности сервера

На вашем компьютере:

```bash
# Проверка доступности
ping 195.58.51.50

# Проверка SSH порта
telnet 195.58.51.50 22
```

### 9. Обращение в поддержку RUVDS

Если ничего не помогает:
- Напишите в поддержку RUVDS
- Укажите номер сервера: RU2289652
- Опишите проблему

## Быстрые действия:

1. **Попробуйте перезагрузить сервер через панель RUVDS:**
   - Зайдите в панель управления
   - Найдите ваш сервер
   - Нажмите "Перезагрузить" или "Reboot"
   - Подождите 2-3 минуты

2. **Попробуйте подключиться через SSH:**
   ```bash
   ssh root@195.58.51.50
   ```

3. **Проверьте статус сервера в панели:**
   - Убедитесь, что сервер включен
   - Проверьте, нет ли проблем с оплатой

4. **Попробуйте веб-консоль снова:**
   - Очистите кэш браузера
   - Попробуйте другой браузер
   - Попробуйте в режиме инкогнито

---

## Если сервер перегружен:

502 ошибка могла возникнуть из-за:
- Нехватки памяти (у вас всего 0.5GB RAM)
- Приложение потребляет слишком много ресурсов

**После восстановления доступа:**
- Проверьте использование памяти: `free -h`
- Оптимизируйте приложение
- Рассмотрите апгрейд сервера

## Что делать после восстановления доступа:

1. Проверьте статус приложения: `pm2 status`
2. Проверьте логи: `pm2 logs sravni-ru`
3. Проверьте использование ресурсов: `free -h`, `htop`
4. Перезапустите приложение: `pm2 restart sravni-ru`
